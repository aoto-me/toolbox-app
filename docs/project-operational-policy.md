# プロジェクト運用方針

## 目的

AIを活用したソフトウェア開発・運用の自動化を安全に検証するための実験的プロジェクト。<br/>
AI支援開発、CI/CDの自動化、依存関係の自動管理などの検証を目的としている。<br/>
そのため、下記ルールのもとで運用する。

### 基本ルール

- 逐一の対話ではなく、計画をして、可能な限りAIを自動で走らせる
- 出力されたコードは必ず全て読む
- コーディングはAIに任せて、手で書かない。修正もなるべく自然言語による指示で直す。
  - CSSの微調整のみ、人の手で直接変更可
- 開発環境・運用プロセスの検証がプロジェクトの主目的で、まだ試験運用であるため、アプリの機能は、万が一データが漏洩したり、壊れても致命的でない内容にとどめること

## ルートディレクトリの.claudeは変更しない

ルートディレクトリの.claudeは変更せず、DevContainer 環境では /workspace/.claude/ で必要な設定を行う。

- /home/node/.claude/ → コンテナ再作成で消える
- /workspace/.claude/ → WSLにマウントされているので永続する

## CLAUDE.md について

- 静的解析やフォーマッターで対応できることを書かない
- 禁止事項は「お願い」ではなく、permissionやhooksのガードで対応する
- 毎回必要でない知識は、.claude/rules にまとめる

## 会話履歴について

開発コンテナをリビルドすると、会話履歴やメモリが消えるため、作業途中の内容や引き継ぐ必要のある内容は、ドキュメントに残す。<br/>

## docs/specs について

このプロジェクトを0からAIエージェントに再構築させたとき、同じアプリが再現できるだけの資料を docs/specs に揃える。<br/>
そのため、docs/specs を常に最新の状態に保つ。
