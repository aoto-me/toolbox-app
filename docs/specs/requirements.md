# 要件定義書

## 1. 目的・背景

### 1.1 プロジェクトの目的

本プロジェクトは、AIを活用したソフトウェア開発・運用の自動化を安全に検証するための実験的プロジェクトである。

主な検証テーマ:

- **AI支援開発**: DevContainer環境でAI（Claude Code等）を安全にコーディングに活用する手法の検証
- **CI/CDの自動化**: GitHub Actionsによるビルド・テスト・デプロイの自動化
- **依存関係の自動管理**: Dependabotによるパッケージの自動更新・自動マージの安全性検証

### 1.2 プロジェクトの位置づけ

- アプリケーションの完成度よりも、開発環境・運用プロセスの構築と検証が主目的
- アプリの機能は、検証の題材として日常的に使うシンプルなツールを採用している
- 万が一データが漏洩したり、アプリが壊れても致命的でない内容に意図的にとどめている

### 1.3 対象ユーザー

- 開発者本人と家族のみ

## 2. 機能要件

### 2.1 認証

- メールアドレスとパスワードによるログイン
- セッション管理（自動期限切れ検知・再ログイン誘導）
- ログアウト

### 2.2 ダッシュボード

- ログイン後に表示される1ページ構成のメイン画面
- 全ツールをグリッドレイアウトで一覧表示
- ツールの並び順をドラッグ＆ドロップで変更可能（順序は端末に保存）

### 2.3 メモ

- テキストメモの入力・自動保存
- 文字数カウント表示（上限1000字）

### 2.4 価格計算

- 2つのモードを切り替えて使用
  - 税込み計算: 金額を入力し、複数税率（8%, 10%）の税込価格を表示
  - 割引計算: 金額と割引率を入力し、割引後価格を表示

### 2.5 ファイルアップロード

- ファイルのアップロード（ドラッグ＆ドロップ対応、複数ファイル可）
- アップロード済みファイルの一覧表示
- ファイルのダウンロード
- ファイルの削除

### 2.6 ブラウザ互換性検索

- CSS/HTML/JavaScript/APIの機能名でブラウザ対応状況を検索
- 入力中のサジェスト表示
- 主要ブラウザ（Chrome, Safari, Firefox, iOS Safari, Chrome Android）の対応バージョンを表示
- MDNドキュメントへのリンク

## 3. 非機能要件

### 3.1 対応環境

- **利用シーン**: スマホとPC間でメモやファイルを手軽に共有するための日常ツール
- **対応ブラウザ**:
  - PC: Google Chrome（最新版）
  - スマホ: iOS Safari（PWA）
- **端末の制約**: iPhone 8（iOS 16.x）を下限とする。このデバイスが対応していないweb機能は使用しない

### 3.2 PWA

- スマホのホーム画面にインストールして、ネイティブアプリのように使用できること
- スタンドアロン表示（ブラウザUIを非表示）
- iOS PWA環境でのプルダウン更新（Pull to Refresh）

### 3.3 セキュリティ

- パスワードはハッシュ化して保存
- ログイン試行のレート制限（ブルートフォース対策）
- セキュリティヘッダーの設定（CSP, HSTS等）
- ファイルアクセスは署名付きURL経由（直接公開しない）
- 依存パッケージのサプライチェーン攻撃対策（公開後14日未満のパッケージのインストール禁止）

### 3.4 監視

- Sentryによるエラートラッキング

### 3.5 開発・運用環境

- DevContainerによるコンテナ化された開発環境の構築
- GitHub ActionsによるCI/CD（lint → test → build → e2e）
- Dependabotによるパッケージ自動更新・自動マージ
