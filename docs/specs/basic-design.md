# 基本設計

## 1. システム構成

### 構成図

```
[ユーザー]
    │ HTTPS (443)
    ▼
[Route: さくらDNS] <本番ドメイン> → Elastic IP
    │
    ▼
[AWS EC2]（ap-northeast-1）
    ├── [Nginx] リバースプロキシ + Let's Encrypt(SSL)
    │       │ proxy_pass → localhost:3000
    │       ▼
    └── [Docker コンテナ] Next.js (toolbox-app)
            │
            ├── [AWS RDS] PostgreSQL 18
            │
            └── [AWS S3]（ファイルストレージ）
                 ※ファイルアクセスは署名付きURL経由

[GitHub Actions] → [Amazon ECR] → SSM経由 → [AWS EC2]

[Sentry]（エラーモニタリング）
```

### 利用サービス一覧

| サービス                    | スペック     | 用途                                       |
| --------------------------- | ------------ | ------------------------------------------ |
| AWS EC2（ap-northeast-1）   | t3.micro     | アプリケーションサーバー（Dockerコンテナ） |
| AWS Elastic IP              | -            | EC2の固定IP                                |
| Nginx（EC2内）              | -            | リバースプロキシ・SSL終端                  |
| Let's Encrypt（Certbot）    | -            | SSL証明書の発行・自動更新                  |
| Amazon ECR                  | -            | Dockerイメージレジストリ                   |
| AWS RDS（PostgreSQL 18）    | db.t4g.micro | データベース                               |
| AWS S3                      | -            | ファイルストレージ                         |
| AWS Systems Manager (SSM)   | -            | EC2への接続・コマンド実行（SSH不使用）     |
| AWS IAM                     | -            | アクセス権限管理（github-actions等）       |
| さくらインターネット（DNS） | -            | ドメイン管理                               |
| Sentry                      | -            | エラーモニタリング                         |
| GitHub Actions              | -            | CI/CD                                      |

### ホスティング選定理由

学習目的でEC2 + Docker + Nginx構成を採用。

## 4. 画面構成・遷移概要

### 画面一覧

| 画面           | パス     | 説明                           |
| -------------- | -------- | ------------------------------ |
| ログイン画面   | `/login` | メール・パスワード入力フォーム |
| ダッシュボード | `/`      | 全ツールを1ページに表示        |
| 404ページ      | —        | 存在しないURLへのアクセス時    |

### 画面遷移

```
[未認証ユーザー]
    │ 任意のURLにアクセス
    ▼
[ログイン画面 /login]
    │ 認証成功
    ▼
[ダッシュボード /]
    │ セッション期限切れ
    ▼
[ログイン画面 /login]（再ログイン後、元の画面に戻る）
```

### ルーティング

- 未認証状態で `/` にアクセスすると `/login` にリダイレクト
- ログイン済み状態で `/login` にアクセスすると `/` にリダイレクト
- ダッシュボードはルートグループ `(dashboard)` で管理

## 5. 環境構成

| 環境             | 用途           | アプリ                                   | DB                                           | ストレージ         |
| ---------------- | -------------- | ---------------------------------------- | -------------------------------------------- | ------------------ |
| ローカル（開発） | 機能開発       | `next dev`（DevContainer）               | Docker Compose（PostgreSQL 16）              | AWS S3（本番共用） |
| テスト（E2E）    | CI・自動テスト | `next build && next start`（ポート3001） | Docker Compose（PostgreSQL 16 / 別コンテナ） | -                  |
| 本番             | 実運用         | AWS EC2（Docker）                        | AWS RDS（PostgreSQL 18）                     | AWS S3             |

### 補足

- ローカル開発のS3は本番と共用（テスト用バケットは未分離）
- E2Eテスト用DBは開発用DBと分離（`db-test:5432/toolbox_test`）
