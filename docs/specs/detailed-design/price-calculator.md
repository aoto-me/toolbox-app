# 詳細設計: 価格計算

## 1. 機能概要

- DBアクセスなし・フロントエンドのみのツール
- タブ切り替えで2つのモードを提供する
- ボタン操作は不要。入力が有効値になった時点で計算結果を即時表示する

### 税込み価格モード

元金額を入力すると、8%（軽減税率）と10%（標準税率）の税込み価格・消費税額を同時表示する。

### 割引価格モード

元金額と割引率を入力すると、割引後価格・割引額を表示する。

## 2. UI要件・操作フロー

### 画面構成

- タブ（「税込み価格」/「割引価格」の切り替え）
- 元金額入力フィールド（単位：円）
- 割引率入力フィールド（単位：%）— 割引価格モード時のみ表示
- 計算結果テーブル — 有効な入力値がそろった時点でのみ表示

### 税込み価格モードの操作フロー

1. 元金額を入力
2. 有効な値（0より大きい数値）になった時点で結果を表示
3. 8%・10%の2行を同時に表示する

表示形式：

```
消費税  8%  |  1,080 円（税 80 円）
消費税 10%  |  1,100 円（税 100 円）
```

### 割引価格モードの操作フロー

1. 元金額を入力
2. 割引率を入力
3. 両方が有効な値になった時点で結果を表示

表示形式：

```
10% OFF  |  900 円（-100 円）
```

### 入力フィールドの仕様

- `type="text"` + `inputMode="numeric"` — スマホではテンキーキーボードを呼び出す
- カンマ区切りの数値も受け付ける（例: `1,000` → `1000` として計算）
- 無効な値（0以下・NaN）の場合は計算結果を表示しない

## 3. 計算ロジック仕様

### parsePrice(input: string): number

カンマ（`,`）を除去してから `parseFloat` で数値に変換する。

| 入力例    | 出力   |
| --------- | ------ |
| `"1000"`  | `1000` |
| `"1,000"` | `1000` |
| `"abc"`   | `NaN`  |

### isValidPrice(value: number): boolean

有効条件：NaN でない、かつ 0 より大きい。

| 入力例 | 結果    |
| ------ | ------- |
| `1000` | `true`  |
| `0`    | `false` |
| `-100` | `false` |
| `NaN`  | `false` |

### isValidDiscountRate(rate: number): boolean

有効条件：NaN でない、かつ 0 より大きく 100 未満。

| 入力例 | 結果    |
| ------ | ------- |
| `10`   | `true`  |
| `0`    | `false` |
| `100`  | `false` |
| `NaN`  | `false` |

### calcTax(base, rate): { tax, taxed }

| 項目                  | 計算式                                |
| --------------------- | ------------------------------------- |
| 消費税額（`tax`）     | `Math.round(base × rate / 100)`       |
| 税込み価格（`taxed`） | `Math.round(base × (1 + rate / 100))` |

端数は四捨五入。適用税率は 8% と 10% の2種類。

### calcDiscount(base, rate): { discounted, saving }

| 項目                       | 計算式                                |
| -------------------------- | ------------------------------------- |
| 割引後価格（`discounted`） | `Math.round(base × (1 - rate / 100))` |
| 割引額（`saving`）         | `Math.round(base × rate / 100)`       |

端数は四捨五入。
