# 詳細設計: メモ

## 1. UI要件・操作フロー

### 画面構成

- テキストエリア（複数行・入力量に合わせて高さが自動伸長）
- フッター：保存ステータス（左）+ 文字数カウンター（右）

### 初期表示フロー

1. サーバーコンポーネント（`page.tsx`）が DB から直接メモを取得
2. `DashboardDataProvider` 経由でクライアントコンポーネントに初期データを渡す
3. `Memo` コンポーネントは `useDashboardData()` で初期値を受け取り、テキストエリアにセット
4. メモ未登録（初回）→ 空文字・更新日時なしで初期化

### 入力・保存フロー

1. ユーザーがテキストエリアに入力
2. 1000文字を超える入力は無視（変化なし）
3. 入力のたびにデバウンスタイマー（1500ms）をリセット
4. 最後の入力から1500ms経過 → Server Action `saveMemo()` で自動保存

### 保存ステータス表示

| 状態                         | 表示テキスト       |
| ---------------------------- | ------------------ |
| `idle`（初期、更新履歴なし） | 空白               |
| `idle`（更新履歴あり）       | 更新日時の相対表示 |
| `saving`                     | 保存中...          |
| `saved`                      | 更新日時の相対表示 |
| `error`                      | 保存に失敗しました |

### 更新日時の相対表示形式

| 経過時間          | 表示例        |
| ----------------- | ------------- |
| 60秒未満          | たった今保存  |
| 60秒〜59分59秒    | 3分前に保存   |
| 1時間〜23時間59分 | 2時間前に保存 |
| 24時間以上        | 5日前に保存   |

## 2. 自動保存の仕組み

### デバウンス処理

- 入力のたびに既存タイマーをクリアして新しいタイマーをセット
- タイマーの待機時間は **1500ms**
- 最後の入力から1500ms無操作で Server Action `saveMemo()` を呼び出す
- タイマーIDは `useRef` で管理（再レンダリングを発生させずに保持）

### Pull to Refresh との連携

ダッシュボード画面には Pull to Refresh 機能がある。未保存の編集中にリフレッシュが走ると入力内容が失われるため、以下でブロックする：

| タイミング                     | 処理                                                       |
| ------------------------------ | ---------------------------------------------------------- |
| 入力開始時                     | `setRefreshBlocked('memo', true)` でリフレッシュをブロック |
| 保存完了時（次のタイマーなし） | `setRefreshBlocked('memo', false)` でブロック解除          |
| コンポーネントアンマウント時   | `setRefreshBlocked('memo', false)` でブロック解除          |

「次のタイマーなし」の判定：Server Action 呼び出しの `finally` ブロックで `timerRef.current === null` をチェックする。保存中にさらに入力があれば新しいタイマーが積まれているため、ブロックを継続する。

### UPSERT による初回・更新の透過的な処理

- メモはユーザー1人につき1レコード（`memos.user_id` に UNIQUE 制約）
- Server Action 側で `INSERT ... ON CONFLICT DO UPDATE` を使うため、フロントエンドは常に同じ関数を呼ぶだけでよい
- 「初回保存か更新か」をフロントエンドが判断する必要はない

## 3. Server Action仕様

メモの初期取得はサーバーコンポーネント（`page.tsx`）が DB から直接取得する。保存はServer Actionで行う。

### `saveMemo(content: string)` — `lib/actions/memo.ts`

**認証**: 必須（内部で `auth()` を呼び出す）

**引数**:

| 引数      | 型     | 説明             |
| --------- | ------ | ---------------- |
| `content` | string | 保存するメモ内容 |

**戻り値**:

```ts
{
  updatedAt: string;
} // ISO 8601形式
```

**エラー（throwされる）**:

| 条件                     | メッセージ                   |
| ------------------------ | ---------------------------- |
| 未認証                   | `ログインが必要です`         |
| `content` が1000文字超過 | `文字数が上限を超えています` |

## 4. バリデーション・制約

| 項目         | 制約                     | 適用レイヤー                                                                               |
| ------------ | ------------------------ | ------------------------------------------------------------------------------------------ |
| 最大文字数   | 1000文字                 | フロントエンド（`maxLength` 属性・入力時チェック）、Server Action（`validateMemoContent`） |
| 認証         | ログイン済みであること   | Server Action（`auth()` チェック）                                                         |
| ユーザー分離 | 自分のメモのみ読み書き可 | Server Action（`session.user.id` でフィルタ）                                              |
| レコード数   | ユーザー1人につき1件     | DB（`memos.user_id` の UNIQUE 制約）                                                       |
